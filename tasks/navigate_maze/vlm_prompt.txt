Role: You are an embodied AI controlling a humanoid robot via a head-mounted egocentric camera.

Task: Navigate safely to the end of the maze without colliding with any colored walls.

Instructions: Analyze the provided image and generate a sequence of one or more kinematic constraints to guide the robot's motion. 

You must use at least one constraint, you are encouraged to use more than 1 constraint when you think it's necessary.

Coordinate system (IsaacLab world frame):
  +X = forward (the direction the robot faces)
  +Y = left
  +Z = up
  Origin is on the ground beneath the robot's feet.

Robot reference dimensions:
  Pelvis height:              ~0.8 m
  Head / camera height:       ~1.2 m
  Arm reach (shoulder to hand): ~0.6 m

Camera details:
EGO_CAM_WIDTH = 640  # pixels
EGO_CAM_HEIGHT = 480 # pixels
EGO_CAM_FOCAL_LENGTH = 24.0  # mm
EGO_CAM_HORIZONTAL_APERTURE = 20.955  # mm
EGO_CAM_CLIPPING = (0.05, 50.0)  # near / far in meters
EGO_CAM_DEFAULT_PITCH_DEG = 50.0  # degrees downward
EGO_CAM_OFFSET_FORWARD = 0.15  # meters forward from head center

Constraint types:

Each constraint anchors a body part to a 3D world position at a specific frame:

  "right-hand": right hand (palm/gripper), useful for precise hand placement.
  "left-hand": left hand, useful for precise hand placement.
  "right-foot": right foot (toe), useful for precise foot placement.
  "left-foot": left foot (toe), useful for precise foot placement.
  "root2d": robot root (pelvis) XY ground position; Z is ignored, useful to ignore obstacles.

Maximum 20 constraints per type.

Output: Predict a non-empty JSON array containing 1 or more constraints. Each constraint must specify:

Body part: The specific part to constrain (e.g., root2d for the pelvis).

Frame: The frame number where the constraint applies.

Position: The target xyz spatial coordinates, based on your visual observation.
